## 📊 데이터 구조

### 표시되는 제품 정보

```typescript
interface Product {
  // 시각적 요소
  coverImage: string;  // 앨범 커버 이미지

  // 기본 정보
  title: string;       // "Side To Side"
  type: 'SINGLE' | 'EP' | 'ALBUM';
  artistName: string;  // "RIIINGS"

  // 식별자
  upc: string;         // "8721465008227"
  ean: string;         // "8721465008227" (동일)
  isrc?: string;       // Track-level

  // 날짜
  addedDate: string;   // "12/8/2025"
  releaseDate: string; // "1/22/2026"

  // 선택 상태
  selected: boolean;   // 체크박스
}
```

## 🎨 UI 요소 분석

### 1. 검색 기능
```
- 위치: 상단 좌측
- 기능: 제품 검색
- Placeholder: "Search products"

```
- 위치: 상단 우측
- 스타일: + 아이콘 + 텍스트

## 🔄 사용자 워크플로우

### 제품 추가 프로세스
```
1. Release Project 페이지 접속
2. "Add products" 버튼 클릭
3. 다이얼로그 열림
4. (선택) 검색으로 필터링
5. 원하는 제품 체크박스 선택
6. "ADD" 버튼 활성화
7. "ADD" 클릭
8. Release Project에 제품 추가됨
```

### 새 제품 생성 프로세스
```
1. "CREATE NEW PRODUCT" 버튼 클릭
2. 새 탭에서 Fillout 폼 열림
3. 제품 정보 입력
4. 제출
5. 제품 목록에 추가됨
```

## 🎨 UI/UX 특징

### 커버아트 표시
- ✅ **모든 제품에 커버아트 이미지 표시됨**
- 썸네일 크기 일정
- 왼쪽에 배치
- 선명하고 명확함

### 텍스트 계층
- **제목**: 큰 글씨, 굵게
- **타입**: 작은 글씨, 회색
- **아티스트**: 중간 글씨, 회색
- **UPC/날짜**: 작은 글씨, 연한 회색

### 선택 메커니즘
- 체크박스로 다중 선택 가능
- 선택 시 "ADD" 버튼 활성화
- 선택 없으면 "ADD" disabled

---

## 💡 N3RVE 플랫폼 적용 포인트

### 현재 N3RVE의 문제
- 커버아트가 표시되지 않음
- 음표 아이콘만 표시

### FUGA가 잘하는 점
1. **커버아트 필수 표시**: 모든 제품에 이미지
2. **명확한 정보 계층**: 제목 > 타입 > 아티스트
3. **검색 + 필터**: 빠른 탐색
4. **페이지네이션**: 대량 데이터 처리
